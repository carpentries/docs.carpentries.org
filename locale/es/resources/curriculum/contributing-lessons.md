# Cómo contribuir a las clases de carpintería

Todas las lecciones de Carpentries están alojadas en GitHub.

Esto le da acceso a una «trastienda» donde el código del sitio web del plan de estudios está disponible de forma gratuita, se pueden proponer sugerencias de cambios y se pueden mantener conversaciones sobre posibles cambios.

Para llegar allí, haz clic en el enlace que dice «Editar esta página» cerca del título de la página, o «Editar en GitHub» en el pie de página en la parte inferior de la página.

! [Edita esta página en Github] (img/edit-this-page.png «la ubicación del enlace Editar esta página en el sitio web de una lección»)

También puedes encontrar enlaces a los repositorios de GitHub para cada lección en las tablas de la página «Lecciones» de cada uno de los programas de lecciones de The Carpentries: [Data Carpentry] (https://datacarpentry.org/lessons/), [Library Carpentry] (https://librarycarpentry.org/lessons/) y [Software Carpentry] (https://software-carpentry.org/lessons/).

! [Tabla de lecciones de carpintería de software] (img/swc-lesson-table.png «tabla de lecciones de carpintería de software en inglés, con los enlaces a los repositorios de lecciones resaltados»)

Los enlaces de la columna _Repositorio_ de estas tablas lo llevarán al repositorio de origen de la lección correspondiente. Hay dos pestañas en esta página que ofrecen oportunidades de interacción con la comunidad que mantiene la lección.

! [Repositorio GitHub de formación de instructores] (img/lesson-repository-homepage.png «Captura de pantalla del repositorio de GitHub de formación de instructores con las pestañas de problemas y solicitudes de cambios resaltadas»)

La primera es la pestaña «Problemas». Cuando alguien tiene una idea o sugerencia, puede iniciar una conversación sobre su idea creando un asunto. Luego, otros pueden comentar sobre el problema para proporcionar comentarios. Los problemas se pueden buscar usando la barra de búsqueda en la parte superior.

! [pestaña de problemas de GitHub Repo] (img/issue-page.png «la pestaña de problemas de un repositorio de GitHub, con la barra de búsqueda y el botón de nuevo problema resaltados»)»

Si tienes una idea, ¡es una buena idea buscar primero para ver si alguien más ya la ha sugerido! Si es así, puedes añadir tus ideas como comentario sobre el tema existente en lugar de iniciar uno nuevo. (De cualquier manera, puedes enviar un enlace a la conversación con tu contribución para [el paso _Participar_ del proceso de formación de instructores] (https://carpentries.github.io/instructor-training/checkout.html#getinvolved)). Ten en cuenta que debes iniciar sesión en GitHub para poder ver el cuadro de comentarios en la parte inferior de cada número.

! [hilo de problemas de GitHub] (img/issue-thread.png «un hilo de problemas en GitHub que resalta la casilla utilizada para añadir un comentario»)

Si encuentras algo que sabes cómo solucionar cambiando el código de la lección, tal vez prefieras enviar un Pull Request (PR). También puede revisar y comentar los RP existentes. Para saber cómo enviar un PR, consulta esta \[guía aportada por la comunidad] [github-guide].

Cada repositorio de lecciones tiene un documento, denominado `Contributing.md`, que proporciona información sobre cómo contribuir a esa lección. (\[Ejemplo: las directrices para el repositorio curricular de formación de instructores] \[contribuir].) Los mantenedores de cada repositorio pueden optar por personalizar sus propias directrices.

El espacio de trabajo de Slack de Carpentries alberga un canal para las personas que contribuyen por primera vez, llamado `#first -time-contributors`. Este es un excelente lugar para presentarse, hacer preguntas y compartir sus logros mientras aprende a contribuir a nuestros proyectos de código abierto.

## Alternativas a GitHub

Aprender a interactuar en GitHub puede tener muchos beneficios, ¡incluida la posibilidad de contribuir a otros proyectos de código abierto! Sin embargo, entendemos que hay muchas razones por las que los alumnos desean evitar participar en GitHub. ¡Eso está bien!

Si no utilizas GitHub, puedes enviar tu contribución a través de [este formulario] (https://docs.google.com/forms/d/e/1FAIpQLSeMBOj5Rdh8Mgk0ebRbeRyHhGHKbItdft6avWuEzmeg8CgWbA/viewform). Un miembro del equipo principal de Carpentries creará una incidencia basada en el repositorio correspondiente y te enviará un enlace para que puedas ver las respuestas.

## Se busca ayuda: Rompiendo barreras

En The Carpentries, sabemos que las herramientas y los procedimientos técnicos pueden representar un serio obstáculo para el progreso.  ¡Por eso existimos! Por eso, apreciamos la ironía de que la participación en nuestra comunidad exija herramientas o procedimientos técnicos que pueden no ser familiares o accesibles para todos los que desean contribuir. GitHub, en particular, es a menudo reconocido como un obstáculo; esto es algo que nuestra comunidad \[discutió ampliamente] [git-blog].

Invertimos activamente en hacer que nuestros procesos sean lo más amigables e inclusivos posible, y eso incluye ayudar a nuestra comunidad a superar las barreras que presentan las tecnologías que elegimos. ¡Mire nuestros canales para conocer las oportunidades de desarrollar sus habilidades como colaborador! Mientras tanto, si tiene alguna idea sobre cómo podemos hacer que nuestros recursos sean más fáciles de usar para los contribuyentes, \[háganoslo saber] \[página de contacto].

### Sumar o restar

Al considerar la posibilidad de contribuir a una lección, ¡ten en cuenta que la mayoría de nuestras lecciones principales ya están repletas de contenido excelente! En muchos casos, las sugerencias sobre cómo reducir o simplificar pueden resultar más útiles que una propuesta de contenido o ejercicios adicionales. Si quieres proponer contenido nuevo, considera dirigir tu contribución a [Glosario] (https://github.com/carpentries/glosario/) o a una lección en desarrollo activo en [The Carpentries Incubator] (https://carpentries.org/community-lessons/).

### ¿Qué viene después?

Después de enviar tu contribución, los mantenedores u otras personas pueden tener preguntas o solicitudes de seguimiento, por lo que debes estar atento a las notificaciones de GitHub. De forma predeterminada, GitHub te notificará sobre cualquier actividad adicional en las conversaciones en las que hayas participado o promovido. Como los encargados del mantenimiento de las lecciones, al igual que los instructores, son en su mayoría voluntarios, ¡es posible que tarden algún tiempo en responder! Algunos repositorios tienen tiempos de respuesta más lentos que otros. No lo tome como algo personal si su contribución no recibe una respuesta inmediata. (No es necesario revisar ni aceptar su contribución para cumplir con el paso «Participar» del proceso de formación de instructores).

## Hacer que su contribución cuente para la certificación de instructor

Si quieres que tu contribución se tenga en cuenta para el requisito de _Participar_ de [Proceso de formación de instructores checkout] [it-checkout], **debes enviarla a través de AMY**. Esto es cierto incluso si asiste a un evento en el que no hay asistencia.

Para esta tarea de pago, haga lo siguiente:

1. inicia sesión en tu \[perfil de AMY] [handbook-amy]. Si no puede iniciar sesión en AMY, consulte a continuación.
2. Cuando hayas iniciado sesión, selecciona «Progreso del entrenamiento» en la parte superior de la pantalla.
3. Selecciona la casilla que mejor describa tu paso para participar en The Carpentries.
4. Introduzca la información solicitada.
5. Su contribución debe evaluarse en un plazo de 7 a 10 días. Le sugerimos que vuelva a comprobarlo para asegurarse de que se ha aceptado su contribución.

Si tiene dificultades para iniciar sesión en AMY, haga lo siguiente:

1. Comprueba que has rellenado el \[formulario de solicitud de formación de instructores] \[solicitud] y que has incluido un nombre de usuario de GitHub en tu solicitud. Tanto el formulario como el nombre de usuario de GitHub son necesarios para que podamos crear un perfil para que puedas iniciar sesión.
2. Envíanos un correo electrónico a [instructor.training@carpentries.org] (mailto:instructor.training@carpentries.org). En este correo electrónico, avísanos cuando hayas completado la solicitud y comparte tu nombre de usuario de GitHub si no lo incluiste anteriormente. También puedes compartir tu selección para el paso _Participar_ para que un miembro de nuestro equipo principal pueda ingresarlo por ti.

### Repositorios aptos

La única regla de elegibilidad para las contribuciones es que deben hacerse en un repositorio administrado por The Carpentries.
Esto incluye (pero no se limita a):

- cualquier lección de [Carpintería de datos] (https://datacarpentry.org/lessons/), [Carpintería de bibliotecas] (https://librarycarpentry.org/lessons/) o [Carpintería de software] (https://software-carpentry.org/lessons/)
- The Carpentries \[plan de estudios de formación de instructores] \[plan de estudios de TI] y \[plan de estudios de desarrollo colaborativo de lecciones] \[plan de estudios cldt-curriculum]
- cualquier \[lección desarrollada por la comunidad] \[incubadora de carpinterías] en The Carpentries Incubator
- nuestro glosario multilingüe de términos de ciencia de datos de código abierto, \[Glosario] [glosario-github]

Los repositorios que **no** reúnen los requisitos para recibir crédito de contribución incluyen bifurcaciones de repositorios de Carpentries y otro contenido basado en Carpentries que no se encuentra en un repositorio de Carpentries. Esto se debe a que el Código de Conducta de Carpentries no se aplica fuera de los espacios de Carpentries. Si envías un enlace a un repositorio que no cumple los requisitos para este paso, es posible que nos pongamos en contacto contigo para solicitarte que redirijas tu contribución.
